# PR Comment Posting — Design Spec

## Overview

After PR analysis completes, automatically post a summary comment on the PR itself (GitHub, GitLab, Bitbucket, Gitea). This is opt-in per repository via a `post_pr_comments` flag on `RepositoryGitConfig`. Each analysis posts a new comment (no editing previous comments), preserving an audit trail across commits.

## Architecture

### Data Flow

```
Webhook received
  → run_pr_analysis() completes (existing)
  → Check config.post_pr_comments
    → True:  format_pr_comment(pr_record) → client.post_comment() → save comment_id/url to PrAnalysis
    → False: skip (current behavior)
```

Commenting is **non-fatal** — failures log a warning but don't change analysis status.

## 1. Platform Client Interface Extension

### `app/git/base.py`

Add `CommentResult` dataclass:

```python
@dataclass
class CommentResult:
    comment_id: str       # Platform-specific comment ID
    comment_url: str      # Direct link to the comment
    platform: str
```

Add abstract method to `GitPlatformClient`:

```python
@abstractmethod
async def post_comment(
    self, repo_url: str, pr_number: int, token: str, body: str
) -> CommentResult:
    """Post a comment on a pull/merge request."""
```

### Platform Implementations

Each platform client implements `post_comment` using their native API:

| Platform   | API Endpoint                                                        | Comment ID field | URL field                  |
|------------|---------------------------------------------------------------------|------------------|----------------------------|
| GitHub     | `POST /repos/{owner}/{repo}/issues/{pr_number}/comments`            | `id`             | `html_url`                 |
| GitLab     | `POST /projects/{id}/merge_requests/{mr_iid}/notes`                 | `id`             | Constructed from project URL |
| Bitbucket  | `POST /repositories/{workspace}/{slug}/pullrequests/{id}/comments`  | `id`             | `links.html.href`          |
| Gitea      | `POST /repos/{owner}/{repo}/issues/{pr_number}/comments`            | `id`             | Constructed from repo URL  |

All use `httpx.AsyncClient` (consistent with existing `fetch_diff` implementations). URL parsing reuses the same `owner/repo` extraction logic already in each client.

## 2. Comment Formatting

### `app/pr_analysis/comment_formatter.py`

Single function:

```python
def format_pr_comment(pr_record: PrAnalysis, base_url: str | None = None) -> str:
```

Takes the completed `PrAnalysis` ORM record. Returns a markdown string.

### Comment Structure

```markdown
## CodeLens Architecture Analysis

**Risk Level:** {emoji} {level} | **Blast Radius:** {n} nodes | **Files Changed:** {n} (+{add} / -{del})

### Impact Summary
- **Changed nodes:** {n} ({breakdown by type})
- **Downstream affected:** {n} nodes
- **Upstream dependents:** {n} nodes
- **Transactions affected:** {list}

### Cross-Technology Impact
- {kind}: `{name}`
...

### Architecture Drift
{warnings about new deps, circular deps}

### AI Analysis
{ai_summary}

---
*Generated by CodeLens • [View full analysis]({url})*
```

Conditional sections:
- **Cross-Technology Impact** — only if `impact_summary.cross_tech` is non-empty
- **Architecture Drift** — only if `drift_report.has_drift` is true
- **AI Analysis** — only if `ai_summary` is non-empty
- **Footer link** — only if `base_url` is provided

Risk level emoji: `High` → red circle, `Medium` → yellow circle, `Low` → green circle.

## 3. Commenter Service

### `app/pr_analysis/commenter.py`

```python
async def post_analysis_comment(
    pr_record: PrAnalysis,
    platform: str,
    api_token: str,
    base_url: str | None = None,
) -> CommentResult:
```

1. Calls `format_pr_comment(pr_record, base_url)` to build markdown
2. Calls `create_platform_client(platform).post_comment(repo_url, pr_number, token, body)`
3. Returns `CommentResult`

Repo URL extracted from `pr_record.pr_url` using the existing `_get_repo_url()` logic in `analyzer.py`.

## 5. Orchestration

### `app/api/webhooks.py` — `_run_analysis_background()`

After `run_pr_analysis()` returns successfully:

```python
# After analysis completes
if post_pr_comments:
    try:
        from app.pr_analysis.commenter import post_analysis_comment
        result = await post_analysis_comment(
            pr_record=pr_record,
            platform=platform,
            api_token=api_token,
            base_url=settings.base_url,
        )
        pr_record.comment_id = result.comment_id
        pr_record.comment_url = result.comment_url
        await session.commit()
    except Exception as exc:
        logger.warning("pr_comment_failed", error=str(exc), exc_info=True)
```

The `post_pr_comments` flag needs to be passed into the background task. Options:
- Pass it as a parameter to `_run_analysis_background()` (simplest, chosen approach)
- Re-query `RepositoryGitConfig` inside the background task
